## Quando usar

Deve ser usando quando o usuário precisa navegaor por uma tela ou executar uma ação específica.

## Diretrizes de uso

- Faça sempre um label de forma clara e compreensível
- Faça sempre pensando sobre a prioridade da ação quando escolher o tipo do botão.
- Faça sempre um botão identificável, na qual o mesmo pode indicar que ele pode desencadear uma ação.
- Faça sempre um botão localizável, o usuário deve ter facilidade de encontrar entre outros elementos o botão.
- Combine diferentes tipos de botão caso em uma tela específica contenha mais de um botão.
- Use o estado de carregamento para fornecer feedback para nosso usuário quando a ação gira em torno de salvar ou inserir informações, em especial se demorar alguns segundos.

## Diretrizes de conteúdo

- Sempre comece a label com um verbo acionável acompanhado por um substativo para fornecer contexto suficiente (e.g. "Confirmar compra"). As únicas exceções são para palavras que são auto-explicativa e que carrega um contexto. (e.g. "Salvar", "Cancelar" e "Fechar").
- A label deve ter no máximo 3 palavras.
- A label não deve ter todas as letras maiúsculas, apenas a primeira palavra ou a palavra da marca/produto (e.g. Google Play Store).

## Variações de componentes

### Variants

O **botão Filled** deve ser usando para as ações mais importantes. Sempre deve haver apenas um **botão Filled** visível por vez na tela.

O **botão Filled Tonal** deve ser usado para ações secundárias para complementar uma ação primária ou quando várias ações de igual importância são necessárias.

O **botão Outlined** deve ser usado para ações secundárias para complementar uma ação primária ou quando várias ações de igual importância são necessárias.

O **botão Text** deve ser usado para ações de apoio e pode ser emparelhado com o botão principal ou secundário.

### Size

O botão oferece suporte a 4 tamanhos diferentes:

- **small**, usado quando há restrições de tamanho.
- **normal**, o tamanho padrão.
- **large**, usado para enfatizar a ação.
